## Bounded Contexts (Ограниченные контексты)

Система разделена на 5 ограниченных контекстов, соответствующих основным разделам приложения:

| Контекст | Ответственность |
|---|---|
| **Контекст планирования** | Ввод исходных данных: проекты, ориентиры, зависимости, ограничения |
| **Анализ портфеля** | Анализ текущего состояния портфеля на основе введённых данных |
| **Стратегические цели** | Формирование и проверка стратегических целей |
| **Альтернативы** | Генерация и сравнение сценариев реализации портфеля |
| **Стратегический план** | Формирование и ведение итогового плана |

## BC: Контекст планирования

Управляет вводом исходных данных, необходимых для всех последующих этапов работы системы.

### Aggregates

| Aggregate Root | Члены агрегата | Описание |
|---|---|---|
| **Стратегические ориентиры** | `Ориентир` | Список ориентиров компании — неделимая единица консистентности: добавление/удаление ориентира меняет агрегат целиком |
| **Реестр проектов** | `Проект` | Перечень проектов мультипроекта; проект существует только в рамках реестра |
| **Зависимости проектов** | `Зависимость` | Список связей между проектами; зависимость не имеет смысла без реестра |

### Entities

| Сущность | Идентификатор | Примечание |
|---|---|---|
| **Проект** | `Название` | Мутабельный объект с жизненным циклом Создание → Удаление |
| **Ориентир** | `Видение` | Конкретный стратегический ориентир с приоритетом |
| **Зависимость** | `Основной проект` + `Зависимый проект` | Направленная связь между двумя проектами |

### Value Objects

| Value Object | Атрибуты | Примечание |
|---|---|---|
| **Данные планирования** | `Название мультипроекта`, `Горизонт планирования` | Неизменяемый контекст мультипроекта; нет собственного ID |
| **Портфельные ограничения** | `Лимит: аналитики`, `Лимит: разработчики`, `Лимит: тестировщики`, `Критический срок` | Определяется набором числовых лимитов; равенство — по значениям |

### Repositories

| Repository | Агрегат |
|---|---|
| **РеестрПроектовRepository** | Реестр проектов |
| **ЗависимостиПроектовRepository** | Зависимости проектов |
| **СтратегическиеОриентирыRepository** | Стратегические ориентиры |

## BC: Анализ портфеля

Вычисляет и хранит результаты анализа текущего состояния портфеля. Читает данные из BC «Контекст планирования» и публикует события для BC «Стратегические цели».

### Aggregates

| Aggregate Root | Члены агрегата | Описание |
|---|---|---|
| **Риски анализа** | `Риск анализа` | Список рисков, выявленных при анализе; формируется и сохраняется как единое целое |
| **Анализ текущих отклонений** | `Отклонение` | Список отклонений по проектам с оценкой опасности |

### Entities

| Сущность | Идентификатор | Примечание |
|---|---|---|
| **Риск анализа** | `Риск` + `Источник` | Конкретный риск с формулировкой, влиянием и источником |
| **Отклонение** | `Проект` | Отклонение привязано к конкретному проекту |

### Value Objects

| Value Object | Атрибуты | Примечание |
|---|---|---|
| **Реализуемость портфеля по срокам** | `Описание` | Результат вычисления — текстовое описание; нет ID |
| **Загрузка и дефицит ресурсов по ролям** | `Роль`, `Суммарная потребность`, `Лимит`, `Дефицит/Профицит` | Расчётный объект для каждой роли |
| **ИИ-объяснения** | `Описание` | Текстовый результат обращения к ИИ; нет ID, привязан к моменту расчёта |
| **ИИ-рекомендации** | `Заголовок`, `Текст` | Результат генерации ИИ; нет ID |

### Domain Services

| Сервис | Что делает |
|---|---|
| **ГрафЗависимостейService** | Строит `Граф зависимостей проектов` по данным `Реестра проектов` и `Зависимостей проектов` |
| **АнализРисковService** | Выявляет риски на основе зависимостей, отклонений, ограничений; формирует `Риски анализа` |
| **АнализОтклоненийService** | Оценивает `Отклонение` каждого проекта и распространение по зависимостям |
| **РесурсыService** | Считает `Загрузка и дефицит ресурсов по ролям` из реестра проектов и портфельных ограничений |

### Repositories

| Repository | Агрегат |
|---|---|
| **РискиАнализаRepository** | Риски анализа |
| **АнализОтклоненийRepository** | Анализ текущих отклонений |
| **РезультатыАнализаRepository** | Реализуемость портфеля по срокам, Загрузка и дефицит ресурсов по ролям, ИИ-объяснения, ИИ-рекомендации |

## BC: Стратегические цели

Управляет стратегическими целями, их проверкой, конфликтами и дублированием.

### Aggregates

| Aggregate Root | Члены агрегата | Описание |
|---|---|---|
| **Список целей** | `Цель` | Основной агрегат контекста; цель существует только в рамках списка |
| **Список конфликтов целей** | `Конфликт целей` | Список выявленных конфликтов; пересчитывается при изменении целей |
| **Дублирование целей** | `Дубль целей` | Список смысловых совпадений между целями |

### Entities

| Сущность | Идентификатор | Примечание |
|---|---|---|
| **Цель** | `Specific` (формулировка) | Мутабельный объект с полным набором SMART-атрибутов |
| **Конфликт целей** | `Категория` + `Список целей` | Конкретный конфликт между двумя и более целями |
| **Дубль целей** | `Категория` + `Цели` | Конкретное смысловое совпадение между целями |

### Value Objects

| Value Object | Атрибуты | Примечание |
|---|---|---|
| **Покрытие проектами** | `Степень покрытия` | Результат ИИ-оценки покрытия цели проектами; нет ID |
| **Проверка реалистичности цели** | `Степень реалистичности`, `Оценка`, булевы флаги по критериям | Вычисляемый объект-оценка; нет ID |
| **Цели, предложенные ИИ** | `Список целей` | Временный объект до добавления целей пользователем в список |

### Domain Services

| Сервис | Что делает |
|---|---|
| **ПроверкаРеалистичностиService** | Оценивает полноту заполнения цели и возвращает `Проверку реалистичности цели` |
| **ПокрытиеПроектамиService** | Определяет степень покрытия цели связанными проектами |
| **КонфликтыЦелейService** | Выявляет конфликты и дублирования в `Список целей`; формирует соответствующие агрегаты |

### Repositories

| Repository | Агрегат |
|---|---|
| **СписокЦелейRepository** | Список целей |
| **КонфликтыЦелейRepository** | Список конфликтов целей |
| **ДублированиеЦелейRepository** | Дублирование целей |

## BC: Альтернативы

Генерирует и хранит альтернативные сценарии реализации портфеля.

### Aggregates

| Aggregate Root | Члены агрегата | Описание |
|---|---|---|
| **Альтернативы** | `Сценарий`, `Риски сценария` → `Риск сценария` | Ровно 3 сценария (Сбалансированный, Консервативный, Рискованный); генерируются и сохраняются единым набором |

### Entities

| Сущность | Идентификатор | Примечание |
|---|---|---|
| **Сценарий** | `Название` / `Тип` | Конкретный вариант реализации портфеля; содержит вложенные `Риски сценария` |
| **Риск сценария** | `Риск` + `Источник` | Риск, специфичный для конкретного сценария |

### Value Objects

| Value Object | Атрибуты | Примечание |
|---|---|---|
| **Матрица сравнения альтернатив** | По каждому сценарию: оценка, риски, ограничения, ресурсы, срок, статус | Производный объект-таблица; нет собственного ID |

### Domain Services

| Сервис | Что делает |
|---|---|
| **ГенерацияСценариевService** | Формирует три сценария на основе реестра проектов, ограничений и зависимостей |
| **ОценкаСценарияService** | Вычисляет интегральную оценку, соблюдение ограничений, ресурсную реализуемость |

### Repositories

| Repository | Агрегат |
|---|---|
| **АльтернативыRepository** | Альтернативы |

## BC: Стратегический план

Формирует и ведёт итоговый стратегический план на основе выбранного сценария.

### Aggregates

| Aggregate Root | Члены агрегата | Описание |
|---|---|---|
| **План** | `Этап` → (`Состав этапа`, `Ресурсы этапа`, `Контрольная точка`), `Риски плана` → `Риск плана` | Единый агрегат с консистентным изменением: правка этапа меняет план целиком |

### Entities

| Сущность | Идентификатор | Примечание |
|---|---|---|
| **Этап** | `Номер этапа` | Часть плана с периодом, составом работ и контрольной точкой; редактируется пользователем |
| **Контрольная точка** | `Название` + `Дата` | Ключевой результат завершения этапа |
| **Риск плана** | `Риск` + `Источник` | Риск, связанный с планом или его этапами |

### Value Objects

| Value Object | Атрибуты | Примечание |
|---|---|---|
| **Паспорт стратегического плана** | `Выбранный вариант`, `Плановый горизонт`, счётчики точек/рисков/ограничений, `Прогресс выполнения` | Сводка плана, вычисляется из данных агрегата; нет ID |
| **Состав этапа** | `Проект`, `Доля`, `Ресурс` | Часть работ по проекту в этапе; определяется набором значений |
| **Ресурсы этапа** | `Роль`, `Требуется`, `Доступно`, `Загрузка`, `Степень загрузки` | Ресурсный расчёт по роли внутри этапа |
| **Состояние этапов планирования** | `Название`, `Статус` | Показывает прогресс прохождения разделов системы; нет ID |

### Domain Services

| Сервис | Что делает |
|---|---|
| **ФормированиеПланаService** | Разворачивает выбранный сценарий в этапы с составом, ресурсами и контрольными точками |
| **РасчётРесурсовЭтапаService** | Считает загрузку ресурсов по этапам и ролям |

### Repositories

| Repository | Агрегат |
|---|---|
| **ПланRepository** | План |

## Domain Events (Доменные события)

| Событие | Источник (BC) | Потребители (BC) |
|---|---|---|
| **КонтекстСформирован** | Контекст планирования | Анализ портфеля — запускает расчёт |
| **АнализЗавершён** | Анализ портфеля | Стратегические цели — разблокирует раздел |
| **ЦельСохранена** | Стратегические цели | Стратегические цели — пересчёт конфликтов, дублирований, покрытия |
| **ЦелиПодтверждены** | Стратегические цели | Альтернативы — разблокирует генерацию сценариев |
| **СценарийВыбран** | Альтернативы | Стратегический план — запускает формирование плана |
| **ЭтапОбновлён** | Стратегический план | Стратегический план — пересчёт паспорта, ресурсов, прогресса |

**См:** [domain-events.md](./domain-events.md)

## Application Services (Сервисы приложения)

Координируют взаимодействие пользователя с доменом, не содержат бизнес-логики.

| Сервис | Ответственность |
|---|---|
| **КонтекстПланированияAppService** | CRUD для данных планирования, ориентиров, проектов, зависимостей, ограничений; инициирует формирование контекста |
| **АнализПортфеляAppService** | Оркестрирует запуск расчётов анализа и сохранение результатов |
| **УправлениеЦелямиAppService** | CRUD целей, запрос ИИ-предложений, добавление предложенных целей в список |
| **АльтернативыAppService** | Запуск генерации сценариев, выбор/снятие выбора сценария |
| **ПланAppService** | Формирование плана из сценария, редактирование этапов |

## Infrastructure Services (Инфраструктурные сервисы)

Реализуют техническую функциональность без доменной логики.

| Сервис | Что предоставляет |
|---|---|
| **ИИService** | Обращения к LLM: формулировка рисков, объяснения, рекомендации, предложение целей, ИИ-интерпретация сценариев и плана, автодополнение полей цели |
| **ГрафВизуализацииService** | Построение визуального графа зависимостей проектов |
| **ИмпортОтчётаService** | Загрузка и парсинг `Отчёта о факторах внешней среды` из файла |
| **БазаДанныхService** | Персистентность всех агрегатов (ORM / query-слой) |
