# Этот файл подробно описывает процесс формирование анализа 

## Формирование графа зависимостей проектов

**Граф зависимостей проектов** Направленный граф с учетом статуса проектов, обычных зависимостей, критической цепочки и зон повышенного риска. Включает в себя:
- Узлы: Проекты.
- Ребра: Зависимости между проектами.
- Атрибуты узлов: Статус проекта (`Завершено`, `В работе`, `Не начато`, `Высокий риск срыва`, `Обычная зависимость`, `Критическая цепочка`)

Процесс формирования графа:

На вход поступают данные о проектах, полученные из БД (эти данные ввел пользователь на этапе формирования контекста).
Входные данные включают в себя:



## Выявляются риски

## Происходит анализ реализуемости портфеля по срокам

## Происходит анализ загрузки и дефицита ресурсов по ролям

## Происходит анализ портфельных ограничений

## Происходит анализ текущих отклонений

## Формируются ИИ-объяснения

## Формируются ИИ-рекомендации

## Все данные сохраняются в БД и становятся доступны на фронтенде для отображения пользователю.
