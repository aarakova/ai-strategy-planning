## Контекст

Термин «событие» в этом документе означает **важный факт в жизненном цикле системы**. Описание каждого события отвечает на вопросы:
- **Где** происходит?
- **Что** запускает это событие?
- **Что происходит** (последовательность шагов)?
- **Что триггерится дальше** (каскадные эффекты)?
- **Побочные эффекты** (логирование, уведомления)?

## События навигации

### Переход на страницу "Контекст"
**Где:** Главная страница

**Триггер:** Нажатие на кнопку "Перейти к формированию контекста".  

**Что происходит:**
- Пользователь переносится на страницу "Контекст"

**Что триггерится дальше:**
- Загружаются ранее сохранённые данные (если есть):
  - Данные планирования
  - Стратегические ориентиры
  - Реестр проектов
  - Зависимости проектов
  - Портфельные ограничения

**Побочные эффекты:** Нет

## События этапа "Контекст"

### Добавление стратегического ориентира
**Где:** Страница "Контекст"

**Триггер:** Нажатие кнопки "Добавить" в блоке "Стратегические ориентиры"

**Что происходит:** 
- Создаётся объект Ориентир
- Выполняется проверка:
  - Видение не пустое
  - Приоритет задан
- Ориентир добавляется в список стратегических ориентиров

**Что триггерится дальше:**
- Обновление списка ориентиров
- Обновление интерфейса

**Побочные эффекты:** Логирование

---
### Удаление стратегического ориентира
**Где:** Страница "Контекст"

**Триггер:** Нажатие кнопки "Удалить" напротив соответствующего ориентира в списке стратегических ориентиров

**Что происходит:**
- Ориентир удаляется из списка

**Что триггерится дальше:** 
- Обновление списка ориентиров
- Обновление интерфейса

**Побочные эффекты:** Логирование

---
### Добавление проекта
**Где:** Страница "Контекст"

**Триггер:** Нажатие "Добавить проект" в блоке "Реестр проектов"

**Что происходит:**
- Создаётся объект Проект
- Выполняется валидация всех полей:
  - название не пустое
  - статус задан
  - дата начала задана
  - дата окончания задана
  - трудоемкость: аналитики не пустое
  - трудоемкость: разработчки не пустое 
  - трудоемкость: тестировщики не пустое 
- Добавляется в Реестр проектов

**Что триггерится дальше:**
- Обновление списка проектов
- Пересчет суммарной трудоемкости в списке проектов
- Обновление интерфейса

**Побочные эффекты:** Логирование

---
### Удаление проекта
**Где:** Страница "Контекст"

**Триггер:** Нажатие "Удалить" напротив соответствующего проекта в `Реестре проектов`

**Что происходит:** 
- Проект удаляется

**Что триггерится дальше:** 
- Обновление списка проектов

- Обновление интерфейса

**Побочные эффекты:** Логирование

### Формирование контекста 
**Где:** Страница "Контекст"

**Триггер:** Пользователь заполнил данные на странице и нажал на кнопку "Сформировать контекст"

**Что происходит:**
- Данные формы отправляются на сервер
- Происходит валидация данных
- Сервер сохраняет данные в БД
- Пользователю возвращается ответ об успешном сохранении данных и формировании контекста

**Что триггерится дальше:**
- Обновление статуса этапа "Контекст"
- Переход на страиницу "Анализ"
- Формирование анализа

**Побочные эффекты:** Логирование, уведомление.

### Формирование анализа
**Где:** Бэкэнд

**Триггер:** Формирование контекста

**Что происходит:**

  - ИИ-объяснения
  - ИИ-рекомендации

**Что триггерится дальше:**
- Сохранение данных в БД
- Данные доступны для отображения на фронтенде на странице "Анализ"

**Побочные эффекты:** Логирование

## События этапа "Цели"

### Добавление цели
**Где:** Страница "Цели"

**Триггер:** Нажатие "Добавить цель" в блоке "Добавление стратегической цели"

**Что происходит:**
- Создаётся объект Цель
- Проверяется:
  - SMART-поля заполнены
  - KPI корректен
- Цель добавляется в `Список целей`

**Что триггерится дальше:**
- Обновление списка целей
- Обновление интерфейса

**Побочные эффекты:** Логирование

---
### Заполнение пустых полей ИИ
**Где:** Страница "Цели"

**Триггер:** Нажатие "Предложить заполнение пустых полей" в блоке "Добавление стратегической цели"

**Что происходит:**
- Передаются данные об уже заполненных полях
- Проверяется:
  - Минимально заполенные поля (`S (Specific) — что необходимо достичь`. `T (Time-bound) — срок достижения`, `Приоритет`)
- Заполняются пустые поля (`M (Measurable) — как будет измеряться достижение цели`, `A (Achievable) — достижимость`)

**Что триггерится дальше:**
- Обновление формы "Добавление стратегической цели"

**Побочные эффекты:** Логирование

---
### Генерация целей ИИ
**Где:** Backend

**Триггер:** Нажатие на кнопку "Перейти к целям"

**Что происходит:**
- В ИИ передаются:
  - анализ
  - проекты
  - ориентиры
- ИИ возвращает список целей

**Что триггерится дальше:**
- Отображение пользователю

**Побочные эффекты:** Логирование

---
### Добавление цели из ИИ

**Где:** Страница "Цели"

**Триггер**: Нажатие "Добавить в основной список"

**Что происходит:**
- Цель переносится в `Список целей`
- Обновляется список целей, предложенных ИИ
- Проставляется пометка "ИИ-предложение"
- Обновляется интерфейс

**Что триггерится дальше:** Ничего

**Побочные эффекты:** Логирование

## События этапа "Альтернативы"

### Генерация альтернатив

**Где:** Backend

**Триггер:** Нажатие на кнопку "Перейти к алтернативам"

**Что происходит:**
- Выполняется расчет сценариев
- Формируются 3 сценария:
  - сбалансированный
  - консервативный
  - рискованный
- ИИ формирует описание, сильные и слабые стороны

**Что триггерится дальше:**
- Отображение пользователю
- Сохранение данных в БД

**Побочные эффекты:** Логирование

---
### Выбор сценария

**Где:** Страница "Альтернативы"

**Триггер:** Выбор сценария по нажатию на кнопку "Выбрать для планирования" в блоке альтернатив или кнопки "Выбрать" в матрице сравнения альтернатив

**Что происходит:**
- Устанавливается Выбор для плана = True
- Отображение выбора в интерфейсе

**Что триггерится дальше:**
- Сохранение выбранной алтернативы в БД
- Возможность формирования плана

**Побочные эффекты:** Логирование
