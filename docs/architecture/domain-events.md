## Контекст

Термин «событие» в этом документе означает **важный факт в жизненном цикле системы**. Описание каждого события отвечает на вопросы:
- **Где** происходит?
- **Что** запускает это событие?
- **Что происходит** (последовательность шагов)?
- **Что триггерится дальше** (каскадные эффекты)?
- **Побочные эффекты** (логирование, уведомления)?

## События навигации

### Переход на страницу "Контекст"
**Где:** Главная страница

**Триггер:** Нажатие на кнопку "Перейти к формированию контекста".  

**Что происходит:**
- Пользователь переносится на страницу "Контекст"

**Что триггерится дальше:**
- Загружаются ранее сохранённые данные (если есть):
  - Данные планирования
  - Стратегические ориентиры
  - Реестр проектов
  - Зависимости проектов
  - Портфельные ограничения

**Побочные эффекты:** Нет

---
### Переход на страницу "Анализ"
**Где:** Страница "Контекст"

**Триггер:** Успешное формирование контекста по нажатию на кнопку "Сформировать контекст". 

**Что происходит:**
- Пользователь переносится на страницу "Анализ"
- Загружаются результаты анализа из БД

**Что триггерится дальше:**
- Обновление статуса этапа "Контекст"
- Отображение аналитических данных

**Побочные эффекты:** Нет

## События этапа "Контекст"

### Добавление стратегического ориентира
**Где:** Страница "Контекст"

**Триггер:** Нажатие кнопки "Добавить" в блоке "Стратегические ориентиры"

**Что происходит:** 
- Создаётся объект Ориентир
- Выполняется проверка:
  - Видение не пустое
  - Приоритет задан
- Ориентир добавляется в список стратегических ориентиров

**Что триггерится дальше:**
- Обновление списка ориентиров
- Обновление интерфейса

**Побочные эффекты:** Логирование

---
### Удаление стратегического ориентира
**Где:** Страница "Контекст"

**Триггер:** Нажатие кнопки "Удалить" напротив соответствующего ориентира в списке стратегических ориентиров

**Что происходит:**
- Ориентир удаляется из списка

**Что триггерится дальше:** 
- Обновление списка ориентиров
- Обновление интерфейса

**Побочные эффекты:** Логирование

---
### Добавление проекта
**Где:** Страница "Контекст"

**Триггер:** Нажатие "Добавить проект" в блоке "Реестр проектов"

**Что происходит:**
- Создаётся объект Проект
- Выполняется валидация всех полей:
  - название не пустое
  - статус задан
  - дата начала задана
  - дата окончания задана
  - трудоемкость: аналитики не пустое
  - трудоемкость: разработчки не пустое 
  - трудоемкость: тестировщики не пустое 
- Добавляется в Реестр проектов

**Что триггерится дальше:**
- Обновление списка проектов
- Пересчет суммарной трудоемкости в списке проектов
- Обновление интерфейса

**Побочные эффекты:** Логирование

---
### Удаление проекта
**Где:** Страница "Контекст"

**Триггер:** Нажатие "Удалить" напротив соответствующего проекта в `Реестре проектов`

**Что происходит:** 
- Проект удаляется
- Удаляются связанные зависимости

**Что триггерится дальше:** 
- Обновление списка проектов
- Пересчет суммарной трудоемкости в списке проектов
- Обновление интерфейса

**Побочные эффекты:** Логирование

---
### Добавление зависимости
**Где:** Страница "Контекст"

**Триггер:** Нажатие "Добавить" в блоке "Зависимости проектов"

**Что происходит:**
- Создаётся объект Зависимость
- Проверяется:
  - проекты не совпадают
  - нет циклической зависимости

**Что триггерится дальше:** 
- Обновление списка зависимостей
- Обновление интерфейса

**Побочные эффекты:** Логирование

---
### Формирование контекста 
**Где:** Страница "Контекст"

**Триггер:** Пользователь заполнил данные на странице и нажал на кнопку "Сформировать контекст"

**Что происходит:**
- Данные формы отправляются на сервер
- Происходит валидация данных
- Сервер сохраняет данные в БД
- Пользователю возвращается ответ об успешном сохранении данных и формировании контекста

**Что триггерится дальше:**
- Обновление статуса этапа "Контекст"
- Переход на страиницу "Анализ"
- Формирование анализа

**Побочные эффекты:** Логирование, уведомление.

## События этапа "Анализ"

### Формирование анализа
**Где:** Бэкэнд
**Триггер:** Формирование контекста
**Что происходит:**
- Формируется: 
  - Граф зависимостей проектов
  - Список выявленных рисков
  - Анализ реализуемости портфеля по срокам 
  - Анализ загрузки и дефицита ресурсов по ролям
  - Анализ текущих отклонений
  - ИИ-объяснения
  - ИИ-рекомендации
**Что триггерится дальше:**
- Сохранение данных в БД
- Данные доступны для отображения на фронтенде на странице "Анализ"
**Побочные эффекты:** Логирование
